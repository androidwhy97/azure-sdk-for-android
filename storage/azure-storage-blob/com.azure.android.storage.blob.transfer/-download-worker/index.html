Create the download worker.

TransferClientwon't call this constructor directly, instead it will enqueue the work along withcriteria to execute it. Theandroidx.work.WorkManagercall this constructor when it identifies thatthe criteria are satisfied (e.g network is available).

If criteria configured for the Worker is no longer met while it is executing, then WorkManager will stop theWorker by callingstop. Such stopped Worker instance will be eventually GC-ed. Whenconstraints are again satisfied and if the Worker was stopped without completion then WorkManager will createand starts new Worker instance.

TransferIdInfoLiveDataCache

A Hash table based cache, with each entry has transferId as 'key' and 'value' as referencetoTransferFlags,and weak reference to.The LiveData pair, i.e. TransferOperationResult LiveData and the TransferInfo LiveDatain a 'value' is related such that, when transferId set on TransferOperationResult LiveDatathen transferInfo of corresponding transfer will start streaming from TransferInfoLiveData.A cache entry will automatically be removed when the TransferInfo LiveData inthat entry's value is no longer in ordinary use. The presence of weak reference to itin the cache entry will not prevent it being collected by the garbage collector.This cache is not synchronized explicitly but implicitly synchronized by enforcing itsmethods to be accessible only from the main thread.The cache is designed to shared across allTransferClientinstances within thesame application process. This enables all Observers of a transfer [e.g. Observers ofupload(tid: 1), Observers of resume(tid: 1)] to listen to same TransferInfo LiveData,irrespective of the TransferClient they used to get the TransferInfo LiveData.The cache implementation assumes that the TransferOperationResult LiveData in the valuedo not strongly refer to the TransferInfo LiveData in the same value eitherdirectly or indirectly, since that will prevent the TransferOperationResult LiveDatafrom being collected.
